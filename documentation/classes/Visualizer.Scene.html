Visualizer.Scene Class

Extends Ember.Object
Defined in: build/js/j-visualizer.js:562

Visualizer Scene Represents the Object used to manage data (Modules), and the way the data's viewed.

The visualization of data is a combined effort of all components contained in this project, accessed through this Object.

clearUnusedViews

() chainable

Defined in build/js/j-visualizer.js:676

clearUnusedViews sends a "clear" request to each ModuleView used by the current instance of Visualizer, which isn't used in any of this scene's widgets. In essence it ensures that any Views that are not currently in use are cleansed.

reload

() Void

Defined in build/js/j-visualizer.js:751

reload initiates the process of updating the visualization by cleaning out old Views and updating all current widgets.

Prevents rapid-exectuion by delaying each request by an amount of time specified by property drawWait, and afterward only using the most recent request (as a debounce).

runWidgets

() chainable

Defined in build/js/j-visualizer.js:706

runWidgets iterated this scene's widgets, and requests that the ModuleView specified for each widget executes the operation specified for each widget.

For example, it may tell one ModuleView, a word cloud, to draw itself, and another ModuleView, a set of icons, to group themselves by common-words.

requestedModuleViews is a computed property that returns a dictionary of Modules -> ModuleViews , used to determine which views will be used by the scene. This is particularly useful when changing scenes to determine which Views need to be cleared out, and which will be used in the next scene.
