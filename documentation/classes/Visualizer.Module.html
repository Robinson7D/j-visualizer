Visualizer.Module Class

Extends Ember.ArrayController
Defined in: build/js/j-visualizer.js:1457

Visualizer Module A Visualizer Module is a collection of data used for Visualization. It keeps access slightly standardized and offers several helpers to simplify data management.

Each Module also has a collection of relevant ModuleViews - Objects that define and take care of how a given Module's data can be displayed.

Defined in build/js/j-visualizer.js:1608

groupedBy creates a cached property on the Module to retrieve its data in the form of a two-dimensional Array, where the sub arrays contain this Module's dataset split into sets that share a common (passed) property.

The outside Array is sorted by length, so the largest groups are at the start.

Since the result is cached, it will only be recomputed when the dataset changes, keeping this efficient. The first time you use groupBy for any given property after the data changes (or is created) runs O(nlogn), but each subsequent access is simply O(1) as it simply grabs the previous data. content

Ember.Array

Defined in build/js/j-visualizer.js:1495

content is the full set of data (in Array , Ember.Array) form that is available to this Module. It is computed based on model and should not be set directly.

Note: this attribute should only be accessed from the data-management side of your app; it's what you generally access for front-end filtering. This Module's ModuleViews should not access the content, however - they should access the dataset (which is a subset of content).

dataset

Ember.Array

Defined in build/js/j-visualizer.js:1524

dataset is the subset of content used for current visualization. It can potentially be the exact same data as content, in full, but it can also be a limited/truncated version (often using the maxLength property). It may also filtered down by properties, etc.

ModuleViews should only look here for their Module data.

maxLength

Integer

Defined in build/js/j-visualizer.js:1483

maxLength is an integer size of how much data should be displayed. Sometimes a project will collect more data than is practical to display to the user (either because it will be confusing, or otherwise too slow); this property represents that limit.

model

Ember.Array

Defined in build/js/j-visualizer.js:1512

model is the array which you should set or append when dumping data. To access data, you should use content or, when applicable, dataset.

moduleViews

Ember.Object

Defined in build/js/j-visualizer.js:1539

moduleViews an object dictionary/map of Visualizer ModuleView objects, each of which must be relevant to this particular Module's data structuring.

For example, if this Module is a set of words with frequencies, a WordCloud may be an applicable ModuleView.
